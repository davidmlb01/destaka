# .impeccable — Design Context — Destaka Dashboard

## Design Context

### Users
Profissionais de saúde (dentistas, médicos, fisioterapeutas) que acessam o dashboard para acompanhar a performance no Google. Chegam após setup de 15 minutos. Não são técnicos. Precisam entender o score e as próximas ações em segundos. Frequência de uso: semanal.

### Brand Personality
Confiável. Automático. Especialista.
3 palavras: **Precisão. Autoridade. Simplicidade.**
Goal emocional: "Meu Google está sendo cuidado por especialistas. Posso focar nos pacientes."

### Aesthetic Direction
- Dark theme exclusivo em verde floresta (#14532D → #0A2E18)
- Âmbar (#D97706 / #F59E0B) como accent de ação e destaque
- Plus Jakarta Sans (display) + Inter (body)
- Referências: Linear app, Vercel dashboard — profissional, limpo, sem ruído
- Anti-referência: dashboards coloridos tipo Monday, Jira

### Design Principles
1. **Score é o herói.** Tudo leva ao número 0-100. Hierarquia clara: score > métricas > ações.
2. **Zero ruído.** Cada elemento deve ter propósito. Sem borders desnecessárias, sem sombras decorativas.
3. **Ação óbvia em 3 segundos.** Ao abrir o dashboard, o profissional sabe exatamente o que fazer.
4. **Consistência é confiança.** Mesmos tokens, mesmos padrões, em toda página.
5. **Mobile funciona.** 60% dos usuários acessam do celular entre pacientes.

### Inconsistências Identificadas (Priorizadas)

**P0 — Crítico:**
1. Layout sidebar duplicado em 6 pages (~600 linhas) — criar `DashboardLayout` component
2. Modal do reviews usa `#0D2B1A` (cor única) — unificar com `rgba(10,46,24,0.98)`
3. Posts usa `#60A5FA` (azul) para "Agendado" — cor não existe em mais nenhum lugar
4. max-width inconsistente (3xl, 4xl, 5xl) sem critério

**P1 — Alto:**
5. `rgba(255,255,255,0.3)` não passa WCAG AA — aumentar para `0.45`
6. Spinner inline duplicado em Reviews e Posts — criar `<Spinner />` component
7. Empty states visuais diferentes em cada página — criar `<EmptyState />` component
8. CompetitorsContent usa `style={{fontSize: X}}` inline — converter para Tailwind

**P2 — Médio:**
9. Nenhum hover/focus state definido nos botões
10. Sem CSS variables/tokens globais
