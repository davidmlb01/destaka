# Story 1.3 — Motor de Otimização Automática
**Epic:** 1 — Core Platform
**Status:** Planejado
**Prioridade:** P0
**Estimativa:** 5 dias
**Depende de:** Story 1.2

## Contexto
Executa automaticamente todas as otimizações que a API do Google permite, sem precisar do usuário. É o principal diferencial da plataforma — o "faz sozinho".

## Acceptance Criteria

- [ ] Lista de ações automáticas gerada junto com o diagnóstico
- [ ] Usuário vê quais ações serão executadas antes de confirmar
- [ ] Botão "Otimizar Automaticamente" dispara o job
- [ ] Ações executadas em sequência com delay (respeitar rate limit)
- [ ] Cada ação salva em `optimization_actions` com status
- [ ] Em caso de falha: logar erro, continuar próximas ações
- [ ] Após execução: novo diagnóstico calculado automaticamente
- [ ] Usuário vê score antes/depois
- [ ] Ações executadas listadas com ícone de sucesso/falha

### Ações automáticas implementadas no V1:
- [ ] Completar horários de funcionamento (se faltando)
- [ ] Corrigir/adicionar categorias secundárias relevantes por segmento
- [ ] Adicionar atributos relevantes por segmento
- [ ] Adicionar/atualizar descrição do negócio (gerada por Claude)
- [ ] Adicionar serviços com descrição (gerados por Claude)

## Tasks

- [ ] Implementar `OptimizationPlanner` (gera lista de ações a partir do diagnostic)
- [ ] Implementar `OptimizationExecutor` (executa cada ação via API)
- [ ] Ações: `updateHours`, `updateCategories`, `updateAttributes`, `updateDescription`, `addServices`
- [ ] Implementar job BullMQ: `auto_optimize_job`

- [ ] Testes: mock da API do Google para cada tipo de ação

## Notas Técnicas
- Delay de 200ms entre chamadas de API para evitar rate limiting
