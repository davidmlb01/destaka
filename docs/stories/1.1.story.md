# Story 1.1 — Google OAuth + Conexão GMB
**Epic:** 1 — Core Platform
**Status:** Done
**Prioridade:** P0 (bloqueante para tudo)
**Estimativa:** 3 dias

## Contexto
Ponto de entrada da plataforma. Sem OAuth funcionando, nada mais funciona. O usuário precisa autorizar o GMM a acessar seu Google Business Profile.

**Nota arquitetural:** Implementado com Supabase Auth (Google provider) em vez de next-auth, pois o schema SQL usa `auth.uid()` em todas as políticas RLS. next-auth quebraria o RLS.

## Acceptance Criteria

- [x] Botão "Conectar com Google" na landing/onboarding
- [x] Fluxo OAuth 2.0 completo com Google (via Supabase Auth)
- [x] Scopes solicitados: `business.manage`, `email`, `profile`
- [x] Access token e refresh token salvos criptografados no banco (AES-256-GCM)
- [x] Após auth: listar todos os perfis GMB do usuário
- [x] Usuário seleciona qual perfil gerenciar
- [x] Perfil salvo em `gmb_profiles` com dados básicos
- [x] Redirecionamento para dashboard após seleção
- [x] Tratamento de erro: OAuth negado pelo usuário
- [x] Tratamento de erro: usuário sem perfil GMB
