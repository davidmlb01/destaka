# Story 1.5 — Dashboard Principal
**Epic:** 1 — Core Platform
**Status:** Planejado
**Prioridade:** P0
**Estimativa:** 4 dias
**Depende de:** Story 1.2

## Contexto
A tela central da plataforma. O usuário vê de um relance o estado do seu perfil, o score, as métricas e as ações pendentes. Design limpo, linguagem acessível.

## Acceptance Criteria

- [ ] Score 0-100 em destaque com gauge visual
- [ ] 6 cards de categoria com score individual e status (verde/amarelo/vermelho)
- [ ] Métricas do Google (cliques, ligações, rotas, views) em cards
- [ ] Gráfico de evolução do score nos últimos 30 dias
- [ ] Seção "Próximas ações" com as 3 mais impactantes
- [ ] Seção "Histórico de otimizações" com últimas ações executadas
- [ ] Botão rápido "Sincronizar agora" (força leitura da API)
- [ ] Indicador de última sincronização
- [ ] Responsivo (mobile-first)
- [ ] Loading states para todos os dados assíncronos

## Tasks

- [ ] Layout base do dashboard (sidebar + main content)
- [ ] Componente ScoreGauge (gauge 0-100 animado)
- [ ] Componentes CategoryCard (6x)
- [ ] Componentes MetricCard (cliques, ligações, rotas, views)
- [ ] Componente ScoreChart (Recharts, últimos 30 dias)
- [ ] Componente NextActionsPanel
- [ ] Componente OptimizationHistory
- [ ] API endpoint: GET /api/dashboard (agrega todos os dados)
- [ ] Loading skeletons para todos os componentes
- [ ] Mobile layout (sidebar colapsável)
